# Contact Messages Feature

## Overview
Store contact form messages in database, add CMS section to view/manage messages with read/unread status, notification badge, and search.

## Database Table

### `messages` Table
```sql
CREATE TABLE messages (
  id INT AUTO_INCREMENT PRIMARY KEY,
  full_name VARCHAR(255) NOT NULL,
  email VARCHAR(255) NOT NULL,
  message TEXT NOT NULL,
  is_read TINYINT(1) DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

| Column | Type | Description |
|--------|------|-------------|
| id | INT | Primary key, auto-increment |
| full_name | VARCHAR(255) | Sender's name |
| email | VARCHAR(255) | Sender's email |
| message | TEXT | Message content |
| is_read | TINYINT(1) | 0 = unread, 1 = read |
| created_at | TIMESTAMP | When message was sent |

## Backend

### New Route: `server/routes/messages.js`

| Method | Endpoint | Description | Auth |
|--------|----------|-------------|------|
| `POST` | `/api/messages` | Save new message (from website) | No |
| `GET` | `/api/messages` | List all messages | Yes |
| `GET` | `/api/messages/unread-count` | Get unread count | Yes |
| `GET` | `/api/messages/:id` | Get single message + mark as read | Yes |
| `DELETE` | `/api/messages/:id` | Delete message | Yes |

### Mount in `server/server.js`
```js
const messageRoutes = require("./routes/messages");
app.use("/api/messages", messageRoutes);
```
